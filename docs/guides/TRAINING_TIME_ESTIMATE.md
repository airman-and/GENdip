# StyleGAN2-ADA 학습 시간 추정

## 현재 설정

- **GPU**: 2개 Quadro RTX 8000 (각 50.75 GB)
- **배치 크기**: 자동 최적화 → 약 128 per GPU (총 256)
- **이미지 크기**: 128x128
- **kimg**: 300 (30 epochs 설정 시)
- **데이터셋**: CelebA (~200k 이미지)

## 시간 계산

### kimg 의미
- **kimg = 300** = 300,000 이미지 처리
- CelebA는 약 200k 이미지이므로, 300kimg ≈ 1.5 epoch

### 처리 속도 추정

#### 배치 크기별 속도 (128x128 이미지)

| 배치 크기 | GPU 개수 | 이미지/초 | kimg/시간 |
|-----------|----------|-----------|-----------|
| 32        | 1        | ~2-3      | ~7-10     |
| 64        | 1        | ~4-5      | ~14-18    |
| 128       | 1        | ~6-8      | ~22-29    |
| 128       | 2        | ~12-16    | ~43-58    |
| 256       | 2        | ~15-20    | ~54-72    |

**현재 설정 (배치 256, 2 GPU)**: 약 **54-72 kimg/시간**

### 총 학습 시간

**300 kimg 학습 시:**
- 최소: 300 / 72 ≈ **4.2 시간**
- 최대: 300 / 54 ≈ **5.6 시간**
- **평균 예상: 약 4-6 시간**

### 더 긴 학습 (고품질 모델)

| kimg | 예상 시간 |
|------|-----------|
| 300  | 4-6 시간   |
| 1000 | 14-19 시간 |
| 2500 | 35-46 시간 (약 1.5-2일) |

## 비교: 다른 설정

### 단일 GPU, 배치 32
- 속도: ~7-10 kimg/시간
- 300 kimg: **30-43 시간** (약 1.5-2일)

### 2 GPU, 배치 128
- 속도: ~43-58 kimg/시간
- 300 kimg: **5-7 시간**

### 현재 설정 (2 GPU, 배치 256) ⭐
- 속도: ~54-72 kimg/시간
- 300 kimg: **4-6 시간** (가장 빠름)

## 실제 학습 단계별 시간

### 초기 단계 (0-50 kimg)
- 네트워크 초기화 및 안정화
- 약간 느릴 수 있음
- 예상: 1-1.5 시간

### 중간 단계 (50-200 kimg)
- 안정적인 학습 속도
- 예상: 2.5-3.5 시간

### 후기 단계 (200-300 kimg)
- 미세 조정
- 예상: 0.5-1 시간

## 체크포인트 저장

- **저장 주기**: 50 ticks (기본값)
- **체크포인트 수**: 300 / 50 = 6개
- 각 체크포인트마다 약 50-60분 간격

## 시간 단축 팁

1. **kimg 줄이기**
   - 300 → 200: 약 3-4 시간
   - 300 → 150: 약 2-3 시간
   - 초기 실험용으로 충분

2. **배치 크기 최대화** (이미 적용됨)
   - 현재 배치 256이 최적

3. **Mixed Precision** (이미 활성화)
   - FP16 사용으로 속도 향상

4. **TF32** (이미 활성화)
   - RTX 8000에서 성능 향상
